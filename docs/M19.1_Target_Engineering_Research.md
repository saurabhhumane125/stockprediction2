# M19.1 Target Engineering Research

==========================================================
## SECTION 1 — CURRENT TARGET
==========================================================
**Target**: `return_5d`
- **Mathematical definition**: $\frac{P_{t+5} - P_t}{P_t}$ (where $P$ is the closing price).
- **Business meaning**: The raw, uncompounded percentage profit or loss realized by holding the asset for exactly 5 trading days.
- **Advantages**:
  - Extremely easy to interpret. A prediction of `0.05` translates exactly to a 5% expected profit.
  - Directly models the business objective (capital growth).
  - No inverse transformations required during inference.
- **Disadvantages**:
  - **Highly Heteroskedastic**: Variance changes drastically across market regimes (e.g., COVID crash vs. bull market). Neural networks struggle to minimize MSE when the target variance is unstable.
  - **Micro-Scale**: Returns are typically bounded between `[-0.10, +0.10]`, resulting in initial MSE losses on the order of $10^{-4}$. This causes vanishing gradients in deep networks.
  - **Fat Tails (Leptokurtic)**: Financial returns exhibit extreme outliers. An unscaled MSE loss function heavily penalizes these outliers, often destroying the gradient flow for the majority of normal trades.

==========================================================
## SECTION 2 — INDUSTRY TARGETS
==========================================================
In quantitative trading, hedge funds, and academic financial machine learning, raw returns are rarely used as direct targets for neural networks. Commonly used targets include:

1. **Log Return**
   - *Definition*: $\ln(\frac{P_{t+5}}{P_t})$
   - *Usage*: Academic standard. Additive across time horizons, symmetric, and slightly reduces the impact of extreme positive outliers compared to simple returns.

2. **Direction (Binary/Ternary)**
   - *Definition*: $1$ if Return > Threshold, else $0$ (or $-1, 0, 1$ for Down/Flat/Up).
   - *Usage*: Retail AI and early quantitative models. Converts the problem to classification (Cross-Entropy Loss), which is far easier for neural networks to optimize.

3. **Excess Return (Alpha)**
   - *Definition*: $Return_{t+5} - Benchmark\_Return_{t+5}$
   - *Usage*: Market-neutral hedge funds. Removes the "beta" (broader market movement), forcing the model to predict the idiosyncratic performance of the specific stock.

4. **Relative Return (Cross-Sectional Rank)**
   - *Definition*: $Rank(Return_{t+5})$ across the entire stock universe for day $t$.
   - *Usage*: Institutional quant funds. Ensures a perfectly uniform, stationary target distribution bounded between `[0, 1]`. Extremely robust to market regimes.

5. **Volatility-Adjusted Return (Risk-Adjusted / Sharpe Target)**
   - *Definition*: $\frac{Return_{t+5}}{Historical\_Volatility_t}$ (or divided by Average True Range).
   - *Usage*: Production quantitative trading systems. Normalizes the target so that a 5% return in a quiet market is treated as a massive signal, while a 5% return in a chaotic market is treated as normal variance.

## SECTION 4 — PRODUCTION ANALYSIS
==========================================================
### 1. Direction (Classification)
- **Problem Solved**: Eliminates the fat-tail MSE penalty. Easy to optimize.
- **New Problems**: Does not predict magnitude. The system could buy a stock that goes up 0.1% and ignore a stock that goes up 15%.
- **Production Complexity**: Low.

### 2. Excess Return (Alpha)
- **Problem Solved**: Protects the model from confusing a general market rally with actual predictive edge.
- **New Problems**: The model no longer predicts absolute direction. A stock might have positive alpha but still lose 5% if the market drops 8%. This is dangerous for a long-only swing trading platform.
- **Production Complexity**: Medium (requires benchmark data during dataset generation).

### 3. Volatility-Adjusted Return
- **Problem Solved**: Directly solves the gradient micro-scale and heteroskedasticity bottlenecks. Target distribution becomes highly stationary and closer to $\mathcal{N}(0,1)$, which is mathematically optimal for deep learning regression.
- **New Problems**: Predictions lose absolute scale. A prediction of `1.5` means the stock is expected to move 1.5x its historical volatility.
- **Production Complexity**: Medium. Requires dividing by historical volatility during dataset generation, and multiplying by current volatility during live inference to recover the actual expected % return.

==========================================================
## SECTION 5 — RECOMMENDATION
==========================================================
**RECOMMENDED TARGET: Volatility-Adjusted Return (`return_5d / rolling_volatility_20d`)**

**Evidence & Justification**:
1. **Mathematical Optimization**: The M18.4 Diagnostics Report identified low prediction variance and unscaled targets as a potential root cause for the GRU's failure to learn. Standardizing the target locally via volatility scaling forces the target distribution closer to $\mathcal{N}(0,1)$, stabilizing gradient flow without needing complex Global Scalers that fail during regime shifts.
2. **Financial Reality**: In swing trading, a 3% return on a highly volatile stock (e.g., TSLA) is noise, whereas a 3% return on a low-volatility stock (e.g., KO) is a massive anomaly. The current `return_5d` target forces the neural network to treat both equally. Volatility scaling forces the network to predict *risk-adjusted statistical anomalies*, which is the true definition of quantitative edge.
3. **Business Alignment**: By predicting volatility-adjusted returns, the model directly optimizes for the Sharpe Ratio.

## SECTION 6 — MIGRATION IMPACT
==========================================================
If Volatility-Adjusted Return is approved, the following migrations are required:

- **Dataset Impact**: `feature_engineering.py` (or `TimeSeriesDataset`) must calculate a new target column: `vol_adj_return_5d = return_5d / (rolling_std_20d + epsilon)`.
- **Tensor Impact**: The NIFTY50 tensors must be regenerated (e.g., `NIFTY50/v4.0`) to embed the new target array `y`.
- **Model Impact**: None. The PyTorch GRU architecture remains identical (Regression, `out_dim=1`), but it will converge much faster due to the stationary target.
- **Inference Impact**: The Inference engine must be updated. When the model predicts `vol_adj_return_5d`, the inference engine must multiply this output by the stock's latest 20-day volatility to produce the final human-readable `expected_return_5d` percentage.
- **Registry Impact**: A new version schema (v4.0) will be registered. `metadata.json` must clearly define the target as `vol_adj_return_5d` to ensure the inference engine triggers the post-processing multiplier.
- **Backward Compatibility**: Any existing v3.0 model in the registry predicting raw `return_5d` will not require the volatility multiplier. The inference engine must use the registry's `metadata.json` target string to branch logic accordingly.
