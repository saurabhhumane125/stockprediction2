## 4. SECTION 3 — DEPENDENCY TRACE
The complete execution path is strictly verified:
1. **Target Generation Path:** `ChronologicalSplitter` / `TensorBuilder` → calls `TargetManager.get_strategy()` → fetches from `StrategyRegistry` → instantiates `LegacyTargetStrategy` → generates target columns cleanly without hardcoded domain knowledge in the builder.
2. **Inference Decoder Path:** `ProductionInferenceEngine` → parses `metadata` hash → calls `TargetManager.get_decoder(self.manifest)` → fetches from `DecoderRegistry` → instantiates `LegacyInferenceDecoder` → maps outputs to semantic business rules.
There are no remaining legacy bypasses in the system.

## 8. SECTION 7 — BACKWARD COMPATIBILITY
Backward compatibility is structurally enforced via fallback semantics:
- **Old metadata / Old inference:** `metadata.get("strategy_name", "legacy")` defaults smoothly to the LegacyInferenceDecoder.
- **Old configuration / Old datasets:** `getattr(config.target, "strategy_name", "legacy")` defaults securely to LegacyTargetStrategy.
- **Old registry entries:** Do not fail loading during registry resolution since `TargetManager` uses graceful `.get(k, default)` behavior.

## 10. SECTION 9 — REMAINING TECHNICAL DEBT
1. **Circular Import Bypass:** `TargetManager.get_decoder()` requires an inline import of `BaseInferenceDecoder` and `DecoderRegistry`. This was implemented to prevent a circular dependency where `manager.py` imported `inference.__init__` which triggered `ProductionInferenceEngine` returning to import `TargetManager`.
2. **Missing ProductionCalibrator:** Other end-to-end tests (like `test_calibrator.py` and `test_pipeline.py`) are currently failing with an `ImportError` because `ProductionCalibrator` was removed from `ml_engine/calibration/calibrator.py` during an earlier, unrelated change.
