## SECTION 2 â€” TRAINING COMMAND
==========================================================
Execute the following command from the repository root:

```bash
python ml_engine/scripts/train_models.py --dataset NIFTY50/v3.0 --experiment M18_GRU_Baseline
```

**Parameters Explained**:
- `python ml_engine/scripts/train_models.py`: Invokes the production entry point which instantiates the `ProductionTrainingRunner` and `TrainingOrchestrator`.
- `--dataset NIFTY50/v3.0`: Directs the pipeline to load the version 3.0 NIFTY50 tensors and metadata.
- `--experiment M18_GRU_Baseline`: Sets the MLflow tracking experiment name, ensuring the run metrics, parameters, and metadata are correctly grouped and searchable in `tracking.db`.

==========================================================
## SECTION 3 â€” EXPECTED PIPELINE
==========================================================
The pipeline will execute strictly in the following order:

1. **Dataset Loading**: Loads `train.pt`, `val.pt`, and `test.pt` tensors into memory.
2. **DataLoader**: Wraps tensors in `TimeSeriesDataset` (applying the correct floating-point dtype) and creates batched PyTorch DataLoaders.
3. **Model**: Instantiates the production `GRUClassifier` architecture with 38 input features.
4. **Training**: Iterates through batches computing MSE loss and performing backpropagation using the AdamW optimizer.
5. **Validation**: Evaluates `val_loss`, `val_rmse`, `val_r2`, and `val_ic` at the end of each epoch without gradient updates.
6. **Checkpoint**: Compares current `val_loss` against the historic best; saves weights to memory/disk if improved.
7. **Early Stopping**: Monitors `val_loss`. Halts the loop if validation loss fails to improve for 15 consecutive epochs.
8. **Best Model Restore**: Reloads the weights from the lowest `val_loss` epoch before final evaluation.
9. **Post Processing**: Generates test predictions (`test_preds.npy`), computes final test metrics, and builds evaluation plots.
10. **Artifact Export**: Serializes the model, scalers, reports, and metrics into a new directory inside `artifacts/candidates/NIFTY50/`.
11. **Registry Attempt**: Attempts to promote the candidate to the local registry (Warning: May fail due to a known `scaler.pkl` vs `feature_scaler.pkl` contract mismatch).

## SECTION 6 â€” FAILURE CHECKLIST
==========================================================
- **CUDA Out of Memory (OOM)**:
  - *Action*: Capture `nvidia-smi` output and PyTorch stack trace.
- **Metric Computation Failure (`ConstantInputWarning` / `NaN` loss)**:
  - *Action*: Capture the `training_report.json` and SQL output of `tracking.db` for the run. Indicates gradients exploded or model collapsed again.
- **Registry Failure (`MissingArtifactError`)**:
  - *Action*: Capture the CLI standard output/error. This is a known orchestration mismatch but does not invalidate the trained model artifacts.
- **Target Dtype Crash**:
  - *Action*: Capture the traceback from `TimeSeriesDataset`. Suggests the commit did not sync correctly to Colab.
