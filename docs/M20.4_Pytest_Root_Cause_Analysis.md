# Pytest Root Cause Analysis

## 1. ERROR CLASSIFICATION

### Category 1: Configuration Errors (Missing Environment Variables)
- **Number of affected files:** 1 (`backend/test_pipeline.py`)
- **First occurrence:** `backend/test_pipeline.py:11`
- **Root cause:** The backend `Settings` class requires `DATABASE_URL` and `SECRET_KEY` environment variables. When pytest runs from the root, it parses `test_pipeline.py` which triggers Pydantic validation before variables are injected.
- **Cascading:** No.

### Category 2: Import Path Failures / Missing Classes
- **Number of affected files:** 4 files directly + 4 duplicates in `release_package`.
- **First occurrence:** `ml_engine/tests/test_automation.py:4`
- **Root cause:** `ml_engine.pipeline.runner` and tests attempt to import `ProductionCalibrator` from `ml_engine.calibration.calibrator`. However, `ProductionCalibrator` has been renamed/replaced by `CalibrationManager` in the source file.
- **Cascading:** Yes. The failure to import in `runner.py` breaks `AutomationManager` and all dependent tests.

### Category 3: Duplicate Test Discovery (Release Package)
- **Number of affected files:** ~42 files
- **First occurrence:** `release_package/backend/tests/api/test_health.py`
- **Root cause:** The `release_package/` directory contains a deployment copy of the repository. Pytest recursively collects these duplicate tests, resulting in `import file mismatch` due to identical module names.
- **Cascading:** No. These are independent test collection errors.

### Category 4: Deleted Legacy References
- **Number of affected files:** 1 (`test_tensor_builder_script.py`)
- **First occurrence:** `test_tensor_builder_script.py:10`
- **Root cause:** This root-level legacy script attempts to import `TargetFactory`, which was deleted and replaced by the Target Strategy Framework (`TargetManager`) in previous milestones.
- **Cascading:** No.

## 2. ROOT CAUSE TREE

```text
Root Cause A: Pytest Misconfiguration (No pytest.ini)
    ├── Error 1: Missing env vars during collection (backend/test_pipeline.py)
    └── Error 2..44: Duplicate test collection in release folder (release_package/*)

Root Cause B: Architecture Regression (Calibrator API Change)
    ├── Error 45: ml_engine/tests/test_automation.py collection fails
    ├── Error 46: ml_engine/tests/test_calibrator.py collection fails
    └── Error 47: ml_engine/tests/test_pipeline.py collection fails

Root Cause C: Legacy Cleanup Incomplete (Target Strategy)
    └── Error 48: test_tensor_builder_script.py collection fails
```
*Note: The 52 errors observed are fully explained by these 3 underlying defects.*

---

## 3. PRIORITY

1. **Root Cause B (Calibrator API): P1 (Core production broken)**. The `PipelineRunner` cannot be imported, breaking all automation and continuous training pipelines.
2. **Root Cause C (Legacy Cleanup): P3 (Legacy cleanup)**. Dead code polluting the workspace.
3. **Root Cause A (Pytest Config): P4 (Test maintenance)**. Restores test visibility.

---

## 4. FIX STRATEGY

**For Root Cause A:**
Create a production-grade `pytest.ini` at the root of the repository.
- Configure `norecursedirs = release_package venv node_modules .git` to prevent duplicate collection.
- Inject default test environment variables (`DATABASE_URL=sqlite:///./test.db`, `SECRET_KEY=test_secret`) using `pytest-env` configuration blocks to satisfy Pydantic during collection.

**For Root Cause B:**
Update `ml_engine/pipeline/runner.py` and `ml_engine/tests/test_calibrator.py`.
- Replace `ProductionCalibrator` imports with `CalibrationManager`.
- Update the instantiation logic inside `PipelineRunner` to match the `CalibrationManager` interface.

**For Root Cause C:**
Delete `test_tensor_builder_script.py`. 
- This is a deprecated integration script. The official tensor building is tested in `ml_engine/tests/test_dataset_builder.py` and `test_tensors.py`.
