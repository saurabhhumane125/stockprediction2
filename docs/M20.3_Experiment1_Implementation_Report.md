**New Files Created:**

**Existing Files Modified:**

## 2. Exact Functions Modified
- **`TargetManager`**: Modified the global module scope to include `import ml_engine.data.tensors.targets.strategies.vol_adjusted` and `legacy` ensuring auto-registration of strategies. Imported the decoders inside `get_decoder()`.
- **`ProductionInferenceEngine.predict()`**: Changed the feature pipeline sequence iteration to capture and pass `raw_sequences` down to `TargetManager.get_decoder(...).decode(raw, current_raw_features, ...)`. Previously, it passed `None`.

## 3. Why Each Change Was Necessary
- **`vol_adjusted.py`**: Isolates the deterministic target definition: `Target = Return_5d / Max(Vol_20d, 1e-8)`.
- **`vol_adjusted_decoder.py`**: A verification check revealed no engineered `volatility_20d` feature exists in the current production pipeline (`ml_engine/data/features/generator.py`). Therefore, the decoder dynamically recalculates `Vol_20d` identically to the target strategy from the raw unscaled close prices. It includes strict defensive programming raising a `ConfigurationError` if `SEQUENCE_LENGTH < 21`.
- **`manager.py`**: Explicit imports trigger the `@StrategyRegistry.register` decorators, populating the registries correctly for runtime resolution.
- **`engine.py`**: Modifying the engine was an **Approved Infrastructure Fix**. A mathematical decoder requires current feature state (prices, volatility, etc.) to un-scale predictions. The old classification pipeline passed `None`, breaking the fundamental `BaseInferenceDecoder` API contract for all future continuous targets.

## 5. Backward Compatibility Verification
- The Legacy Classification Strategy remains unmodified.
- Inference engines passing older Metadata payloads safely default to `"strategy_name": "legacy"`.
- Modifying `engine.py` to pass raw features has zero impact on `LegacyInferenceDecoder`, which natively ignores the `current_features` parameter.

## 6. Statistical Validation Verification
The `validate_volatility_target.py` script was updated to perform a scientifically valid continuous vs continuous comparison (`Raw Return_5d Target` vs `Vol-Adjusted Target`), executed on 50 tickers representing 99,131 aligned sequences.

- **Raw Return_5d Target:** 
  - Mean: 0.0041, StdDev: 0.0443, Kurtosis: 7.24
- **Vol-Adjusted Target:** 
  - Mean: 0.2341, StdDev: 2.6064, Kurtosis: 4.74

**Conclusions:**
The volatility-scaling successfully bounds the target range and significantly suppresses the "fat tails" of the distribution. Kurtosis drops from 7.24 (Raw) to 4.74 (Adjusted). The standard deviation normalizes around `~2.6`, meaning extreme raw market crashes are intelligently penalized rather than dominating the MSE loss landscape.

## 7. Risks
- **Inference Feature Alignment:** The decoder extracts the `"close"` index from `current_features` using the metadata schema. If `metadata.json` goes out of sync with the raw tensor layout, the decoder will scale using the wrong feature column.

## 8. Remaining Technical Debt
- **Calibrator Architecture:** `ProductionCalibrator` remains disconnected across the pipeline. This does not block target engineering or this experiment, but blocks confidence scoring down the line.
