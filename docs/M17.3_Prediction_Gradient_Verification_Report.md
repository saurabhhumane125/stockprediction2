## SECTION 1 — EXECUTIVE SUMMARY

Did the model collapse?
Status: **NOT CONFIRMED** (Disproved).

**Support**: The model did not collapse due to gradient failure, vanishing gradients, or scale mismatch. Our numerical investigation shows that gradients are perfectly healthy (norm ≈ 4.7), weights update effectively, and learning progresses identically to any healthy neural network. 

The true root cause is a **data truncation bug** inside `ml_engine/training/utils.py`. The `TimeSeriesDataset` class hardcodes a cast of all target labels to integers (`torch.long`). Since the regression targets are fractional stock returns (e.g. 0.015, -0.02), casting them to `long` rounds them all exactly to `0`. 

The neural network did exactly what it was optimized to do: it successfully learned to predict an array of perfect zeroes, because its training labels were literally a constant array of zeroes.

## SECTION 5 — CONSTANTINPUTWARNING INVESTIGATION
==========================================================
- **Trace exactly**: `metrics_registry.py` → `spearmanr(y_true, y_pred)`
- **Which array caused ConstantInputWarning?** BOTH arrays.
- **Evidence**: `ml_engine/training/utils.py` contains:
  ```python
  class TimeSeriesDataset(Dataset):
      ...
      self.y = torch.tensor(y, dtype=torch.long)
  ```
  Since `y` contains floating returns bounded roughly inside `[-0.15, +0.27]`, casting them to `torch.long` (int64) truncates them perfectly to `0`. 
  Inside `training_pipeline.py`'s `_train_epoch`, the script calls:
  `y_batch = y_batch.float().to(device)`
  This converts the integer zeroes into float `0.0`s. 
- **Target variance**: Exactly `0.0`.
- **Prediction variance**: Since targets are all zero, the neural network optimizes quickly toward outputting exactly `0.0` to minimize MSE. By the end of Epoch 0, predictions are `0.0` and standard deviation is `0.0`. Both inputs to Spearman have zero variance.

## SECTION 6 — R² INVESTIGATION
==========================================================
- **Trace R² computation**: `R² = 1 - (SS_res / SS_tot)`
- **SS_res**: Computed on predictions of `0.0` against targets of `0.0` = `0.0`.
- **SS_tot**: Variance of `y_true` (which is `0.0`) multiplied by N = `0.0`.
- **Computed R²**: `1 - (0 / 0)`. Sklearn's mathematical fallback when `SS_tot == 0.0` is to return `0.0`. This is the reason R² was exactly 0.0.
