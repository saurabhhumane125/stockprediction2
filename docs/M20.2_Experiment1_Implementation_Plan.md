# Experiment 1: Volatility-Adjusted Return Target

## 1. Mathematical Definition

### Evaluated Formulas
1. **Sharpe-style / Z-Score Return:** `(Return_5d - Mean(Return_20d)) / StdDev(Return_20d)`
2. **Simple Volatility-Scaled Return:** `Return_5d / StdDev(Daily_Returns_20d)`
3. **Annualized Volatility-Scaled Return:** `Return_5d / (StdDev(Daily_Returns_20d) * sqrt(252))`

### Recommended Formula
**Simple Volatility-Scaled Return**

`Target = Return_5d / Vol_20d`

Where:
- `Return_5d` = `(Close_t+5 - Close_t) / Close_t`
- `Vol_20d` = `Standard Deviation of daily returns over the preceding 20 days (t-19 to t)`

We will use the standard rolling 20-day standard deviation (representing approximately one trading month) as the scaling factor. We avoid annualizing the volatility (multiplying by `sqrt(252)`) to keep the target closer to a standard normal scale (since the numerator is a 5-day return and the denominator is a daily standard deviation, the ratio naturally yields values typically in the range of -3 to +3).

## 2. Rationale for this Project
The previous learning diagnostics indicated that the GRU model struggles to generalize across different market regimes. Raw returns are highly heteroskedastic: a 5% return during a market crash means something completely different than a 5% return during a low-volatility bull market. 
By normalizing the target by local volatility:
- The loss function (MSE) is no longer dominated by high-volatility periods.
- The model is forced to predict the *directional conviction* relative to current market conditions, rather than absolute price magnitudes.
- It directly isolates the alpha generation capability from pure beta exposure.

## 3. Files Requiring Modification
Since the Target Strategy Framework is fully decoupled, we will not modify any existing core logic. We will only create new files and append to the registries.

**New Files:**
1. `ml_engine/data/tensors/targets/strategies/vol_adjusted.py`
   - Implements `VolatilityAdjustedTargetStrategy`.
   - Calculates the rolling 20-day volatility and scales the future return.
2. `ml_engine/inference/decoders/vol_adjusted_decoder.py`
   - Implements `VolatilityAdjustedInferenceDecoder`.
   - Reverses the scaling using the current 20-day volatility to output actual predicted returns.
3. `ml_engine/tests/test_vol_adjusted_strategy.py`
   - Unit tests validating the mathematical scaling and inverse decoding.

## 4. Dataset Versioning
- **Current Baseline:** `v1.0` (or equivalent legacy dataset).
- **Experimental Dataset:** We will generate a distinct dataset version, e.g., `EXP_M20_VOL_ADJ/v1.0`. 
- **Isolation:** Because the dataset version determines the storage path in `DATASET_STORAGE_ROOT`, generating this new dataset will completely isolate it from the production baseline. The training pipeline will point to this new version without touching the old tensors.

## 5. Metadata Changes
The existing `MetadataGenerator` will automatically capture the new properties from the strategy interface. The resulting `metadata.json` will contain:
- `"strategy_name": "volatility_adjusted"`
- `"strategy_version": "1.0"`
- Target columns will explicitly reflect the scaling, e.g., `["return_5d_vol_adj"]`.

No changes to the `metadata.py` code are required.

## 6. Inference Decoder Behaviour
The neural network will output a scalar representing the Volatility-Adjusted Return. To present a mathematically meaningful prediction to the trading engine or UI, the decoder must un-scale this value.

**Mechanism:**
1. `TargetManager.get_decoder()` will read `"volatility_adjusted"` from the model's manifest and load the `VolatilityAdjustedInferenceDecoder`.
2. The `decode()` method will receive the `raw_prediction` and `current_features`.
3. It will extract the 20-day historical volatility from `current_features` (either by calculating it dynamically from the close prices in the sequence, or by reading a pre-engineered `volatility_20d` feature column).
4. `Predicted_Return_5d = raw_prediction * current_volatility_20d`.
5. The decoder returns the un-scaled return alongside the raw confidence score.

## 7. Testing Strategy
1. **Mathematical Unit Tests:** Pass synthetic price series into `VolatilityAdjustedTargetStrategy` and assert that the target exactly matches manually calculated `return_5d / rolling_std_20d`.
2. **Decoder Reversibility Test:** Assert that `decoder.decode(strategy.generate(df))` successfully recovers the original un-scaled `return_5d` (within float precision).
3. **Pipeline Integration Test:** Execute a dry-run of `TensorBuilder` with `strategy_name = "volatility_adjusted"` on 100 rows to ensure no schema or NaN-handling crashes occur.

## 8. Rollback Strategy
Because the architecture strictly uses the Strategy Pattern and Dependency Injection:
1. **Zero Code Reverts:** We do not need to delete or revert any code. The `VolatilityAdjustedTargetStrategy` simply lives in the registry alongside `legacy`.
2. **Configuration Revert:** To rollback to the baseline, we change the `TrainingConfig` back to `"strategy_name": "legacy"` and `"dataset_version": "v1.0"`.
3. **Artifact Independence:** Models trained on the experimental target will carry the new strategy metadata. The inference engine will dynamically load the correct decoder, meaning experimental models and legacy models can coexist in production simultaneously without interference.
